# MI100 toolchain

The pinned Mojo wheel does not expose AMD CDNA1/gfx908. This repository keeps
the small source patch in `tools/modular-gfx908.patch`; optimizer and dose code
remain backend-neutral and require no MI100-specific branches.

Reference sparse reductions use fixed 32-lane numerical groups rather than the
hardware warp width. NVIDIA executes one group per warp; MI100 executes two
independent groups per 64-lane wavefront. This keeps Float64 summation
association backend-neutral and uses both halves of the MI100 wavefront.

The validated package was built from Modular commit
`c71933db1b22b6a91749debb0715ce60398a461c`:

```bash
git apply /path/to/trip-mojo/tools/modular-gfx908.patch
./bazelw build //mojo/stdlib/std:std
```

Install the resulting `bazel-bin/mojo/stdlib/std/std.mojopkg` outside this
repository. On the validated cluster it is
`${ROOT}/mi100-toolchain/std.mojopkg`. The expected SHA-256 is
`da280f4d1d5023c0d21f35cf543d81a7a890b8a6fd6fec54e8c91f77808401de`.
Build the shared library with the custom `std.mojopkg` and the wheel's matching
`layout.mojopkg`, then submit plans with `submit.sh --vendor amd`. The submission
path keeps the image's `/opt/rocm/lib` and `/opt/rocm/lib64` paths alongside
Apptainer's injected `/.singularity.d/libs`; replacing that search path hides
`libamdhip64.so` before the workload starts.

Hydra job 22489 reduced the MI100 optimizer from 12.520 s to 6.437 s with fixed
32-lane groups, but the device-bootstrap path stopped at iteration 167 instead
of 271. A matched H200 trace showed identical initial objective and exact step;
the first difference was the bootstrap gradient. Its sparse transpose uses
unordered Float64 atomics, so vendor-dependent last bits changed `complexminp`
RNG decisions immediately.

MI100 reference mode therefore keeps TRiP's Float64 matrix construction and
initial gradient on the host. GPU matrix construction uses vendor-specific
`exp` implementations; its small coefficient differences immediately change
`complexminp` decisions. The canonical matrix is packed once, while all regular
optimizer evaluations and clinical dose run on gfx908. This is a correctness
boundary, not a changed stopping rule.

Hydra job 22559 validated this path end to end. It stopped at iteration 271 and
wrote both RSTs and the DVH byte-identically to `trip_temp`. The MI100 optimizer
took 35.097 s, `OptCmd` 50.51 s, `DoseCmd` 17.47 s and the TRiP process wall
time was 111.845 s. Maximum resident host memory was 23,447,440 KiB. Physical
and biological dose had identical support, with relative L2 errors of
`1.17e-10` and `4.40e-12` and maximum absolute differences of `2.98e-8` and
`3.73e-9` Gy. The runner rejects TRiP errors and fails unless the RSTs and DVH
are byte-identical.

Portable deterministic device matrix construction and bootstrap remain
performance tasks; they are not required for the validated MI100 backend.

The custom package is a temporary compatibility layer. Remove it when upstream
Mojo exposes gfx908 directly.
