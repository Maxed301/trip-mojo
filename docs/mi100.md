# MI100 toolchain

The pinned Mojo wheel does not expose AMD CDNA1/gfx908. This repository keeps
the small source patch in `tools/modular-gfx908.patch`; optimizer and dose code
remain backend-neutral and require no MI100-specific branches.

The validated package was built from Modular commit
`c71933db1b22b6a91749debb0715ce60398a461c`:

```bash
git apply /path/to/trip-mojo/tools/modular-gfx908.patch
./bazelw build //mojo/stdlib/std:std
```

Install the resulting `bazel-bin/mojo/stdlib/std/std.mojopkg` outside this
repository. On Hydra it is
`/lustre/bio/mdick/CUDA/mi100-toolchain/std.mojopkg`. The expected SHA-256 is
`da280f4d1d5023c0d21f35cf543d81a7a890b8a6fd6fec54e8c91f77808401de`.
`run_mojo_mi100.sh` combines that package with the wheel's matching
`layout.mojopkg`, rebuilds the current source for gfx908, and runs the focused
optimizer and clinical-dose ABI tests. Apptainer's injected ROCm libraries must
be present on `LD_LIBRARY_PATH` as `/.singularity.d/libs`.

The custom package is a temporary compatibility layer. Remove it when upstream
Mojo exposes gfx908 directly.
